# Pokyny pro prispivani

Dekujeme za zajem prispivat do naseho projektu. At uz jde o hlaseni chyby, novou funkci, opravu nebo doplnujici dokumentaci, velmi si vazime zpetne vazby a prispevku od komunity.

Pred odeslanim issue nebo pull requestu si prosim prectete tento dokument, abychom meli vsechny potrebne informace pro efektivni reakci na vase hlaseni chyby nebo prispevek.

## Hlaseni chyb a pozadavku na funkce

Pro hlaseni chyb nebo navrhy novych funkci muzete pouzit GitHub issue tracker.

Pri zakladani issue prosim zkontrolujte existujici otevrene nebo nedavno uzavrene issues, abyste se ujistili, ze stejny problem uz nenahlasil nekdo jiny. Snazte se uvest co nejvice informaci. Velmi uzitecne jsou zejmena:

* reprodukovatelny testovaci pripad nebo posloupnost kroku,
* verze naseho kodu, kterou pouzivate,
* vsechny upravy, ktere jste provedli a ktere s chybou souviseji,
* cokoli neobvykleho ve vasem prostredi nebo nasazeni.

## Prispivani pres pull requesty

Prispevky pres pull requesty jsou velmi vitane. Nez nam pull request poslete, ujistete se prosim, ze:

1. Pracujete nad nejnovejsim zdrojovym kodem ve vetvi *main*.
2. Zkontrolovali jste existujici otevrene a nedavno sloucene pull requesty, abyste se ujistili, ze problem uz nevyresil nekdo jiny.
3. Otevrel/a jste issue k prodiskutovani vyznamnejsi prace; neradi bychom, aby vas cas prisel nazmar.

Postup pro odeslani pull requestu:

1. Forknete repozitar.
2. Upravte zdrojovy kod; soustredte se prosim na konkretni zmenu, kterou prispivate. Pokud zaroven preformatujete cely kod, bude pro nas obtizne soustredit se na vasi zmenu.
3. Ujistete se, ze lokalni testy prochazeji.
4. Commitnete zmenu do sveho forku s jasnymi commit messages.
5. Poslete nam pull request a odpovezte na vychozi otazky v rozhrani pull requestu.
6. Sledujte pripadna automaticka selhani CI nahlasena v pull requestu a zustante zapojeni do konverzace.

GitHub poskytuje dalsi dokumentaci k [forkovani repozitare](https://help.github.com/articles/fork-a-repo/) a [vytvoreni pull requestu](https://help.github.com/articles/creating-a-pull-request/).

## Hledani prispevku, na kterych pracovat

Prochazeni existujicich issues je dobry zpusob, jak najit neco, na cem muzete prispivat. Nase projekty standardne pouzivaji vychozi GitHub labels (enhancement/bug/duplicate/help wanted/invalid/question/wontfix), takze issues oznacene jako `help wanted` jsou dobre misto, kde zacit.
