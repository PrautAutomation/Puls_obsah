Motion layer
Pulse Components

Kopírovatelné komponenty pro vývojáře. Každý blok obsahuje živou ukázku, pravidlo použití a HTML snippet. Kompletní pravidla a mapování na views viz PULS-pulse-system-pravidla.md.

🔴 Beacon · max 1 na viewport
Dominantní sonarový efekt — plný kód v samostatném souboru
Hero vizuál, onboarding moment, celebration (dokončení tieru). Plné ringy, sweep, core, glow. Dominuje viewportu. Appka: sonarová varianta. Landing: dosud nedefinováno — bude vycházet z loga, ne radar estetika.
Živá ukázka + kompletní CSS/HTML: component-pulse-beacon.html → sekce Motion layer: Signal Beacon / Pulse System. Obsahuje interaktivní ovládání (tempo, síla, dosah) a plný beacon s ringy, sweep, core, map a readout. Nepoužívej bez konzultace pravidel — Beacon smí být max 1 na viewport a pokud je přítomný, Halo ustupuje.
🟡 Halo · max 1 na viewport
CTA s dýchajícím halo
Jedna dominantní akce v daném view. 3–5 cyklů po scroll-into-view, pak ztichne. JS přidává is-halo-active jen po dobu cyklů. Nikdy infinite loop. Pokud je na stránce Beacon, Halo ustupuje do Quiet.
<button class="pulse-cta" data-pulse="halo" data-halo-cycles="5"> Zjisti, kde začít </button>
🟡 Halo quiet · po 5 cyklech
CTA po ztlumení
Třída is-halo-quiet skryje glow. Přidává ji JS IntersectionObserver po N cyklech. Manuálně přidej pro default-quiet stav.
<!-- Tichý stav (žádný glow) --> <button class="pulse-cta is-halo-quiet" data-pulse="halo" data-halo-cycles="5"> Ztlumené CTA </button>
🟢 Ping · max 3 na viewport
Live status heartbeat
Online · vysílá
Malý puls = aktivní služba, ne statický label. Loop OK (periferní). Text nese informaci, dot je dekorativní. prefers-reduced-motion zastaví jen animaci.
<span class="pulse-status"> <span class="pulse-dot" aria-hidden="true"></span> Online · vysílá </span>
🟢 Ping
Badge s výstupním pingem
🔔3
Upozorní na změnu, nesoutěží s navigací. Wrap element (.pulse-bell) je kontejner pro pozicování badge. Číslo a aria-label nesou informaci, ring je dekorativní.
<span class="pulse-bell" aria-label="3 nová upozornění"> <span aria-hidden="true">🔔</span> <span class="pulse-badge">3</span> </span>
🟢 Ping
Měřák se špičkou signálu
Sync82%
Běžící proces: sync, upload, scan, health check. Fill width nastavíš přes inset (right: 18% = 82% fill). Progressbar a text nesou informaci, ping je dekorativní.
<div class="pulse-meter"> <div class="pulse-meter__top"> <span>Sync</span><strong>82%</strong> </div> <div class="pulse-meter__track" role="progressbar" aria-label="Sync" aria-valuemin="0" aria-valuemax="100" aria-valuenow="82"> <span class="pulse-meter__fill"></span> <span class="pulse-meter__ping" aria-hidden="true"></span> </div> </div>
⚫ Quiet core · default
Žádný pulse = 90 % UI
Karty, dokončené questy, přečtený obsah, navigace, statický text, dokončené tiery, proběhlé sessions. Tma dýchá.
Nemusíš přidávat nic. Quiet core je absence pulse. Viz sekce Quiet Core výše — to je tvůj default pro všechno ostatní.
Hierarchie: Beacon > Halo > Ping > Quiet. Max 1 Beacon + 1 Halo + 3 Pingy na viewport. Dvě stejné úrovně vedle sebe se ruší.

Beacon (hero vizuál, celebration): dosud nedefinovaný pro landing — bude vycházet z loga. V appce sonarová varianta, viz component-pulse-beacon.html sekce Motion.

CSS: Všechny pulse styly jsou v bloku /* Pulse components — copy this block */ v hlavičce tohoto souboru. Zkopíruj celý blok do svého projektu.

JS: Pro Halo 3–5 cycle chování přidej IntersectionObserver — viz <script> na konci tohoto souboru.
Implementation checklist:
1. Zkopíruj pulse tokeny, CSS blok a Halo JS z tohoto souboru.
2. Halo používej přes data-pulse="halo" a data-halo-cycles="3-5"; třídu is-halo-active nastavuje jen JS.
3. Dodrž limity: max 1 Beacon, 1 Halo a 3 Pingy ve viewportu.
4. Každý dekorativní pulse prvek označ aria-hidden="true"; informace musí být i v textu, čísle nebo ARIA popisu.
5. Při prefers-reduced-motion zůstává informace viditelná, mizí jen animované ringy a glow.
Detail page system
Day Content View Components

Tady se skládá detail dne: hero, obsahové tasky, quest sidebar a glossary vrstva. Rozdíl mezi quiet core a selected/done stavy musí být dobře vidět, ale nesmí přepadnout do overdesignu.

DayHeader
Hero detail dne
Speciální header pattern se silnější brand expresí. Musí být stručný, operativní a bez opakování stejné informace v několika vrstvách.
Header detailu dne má nést jen jednu navigační vrstvu, jednu stručnou větu o dnešku, jedno meta pásmo a jednu jasnou primary akci. Neopakuj den kurzu, výstup ani stav dne v dalších dekoračních blocích.
TaskCard
Obsah dne
Tichá obsahová karta. Volitelné listy a code block.
Hover je schválně hezký a živý jako v referenci. Žádný barevný pruh jako default, ale ani mrtvá karta bez reakce.
